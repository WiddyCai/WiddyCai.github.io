Hexo 搭建过程中遇到的坑

搭建原因与此文内容

首先可以先说说为什么使用hexo
在我的角度看：
1.可以套用主题
2.操作不用太繁琐
3.与GitHub挂钩
概念，功能，教程，网上也有蛮多介绍的。再此不一一说明了。
再此是讲讲自己在搭建的过程中遇到的各种坑的事情(:зゝ∠)

<一>有的主题更换无效

主题更换方法是:
  装搭好环境(node.js、git、hexo)之后,在自己本机的博客文件夹下（如：D:\blog_config.yml）修改_config.yml中的theme: 后加主题的名字（如：theme: oishi-master）。

注意点：在修改theme:时，冒号（：）之后一定要敲空格。

<二>想要修改页面的样式格式

  主题的样式文件一般都在样式文件夹的source文件夹下，想要修改时，打开hexe server，登录本地服务器，右键检查样式，找对应的html代码属性或者ID复制，然后再代码编辑器中用Ctrl+F在source文件夹下来查找与之对应的代码，然后修改。

<三>用git生成ssh密钥与公钥时

输入：$ ssh-keygen -t rsa时
会出现：Generating public/private rsa key pair.
Enter file in which to save the key (/c/Users/**/.ssh/id_rsa):
按回车！按几个回车它就生成了，然后与GitHub绑定。

总结：搭建中遇到最大的困难就是修改样式，后续找到相应的代码了，问题就解决了O(∩_∩)O~~
